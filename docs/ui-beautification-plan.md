# UI 美化实施方案

## 目标

将 MapViewer 的界面从当前的"功能原型"升级为一个视觉上精致、交互上流畅的地图应用。只修改 `map-app/src/index.html` 中的 CSS 和 HTML 结构，JS 逻辑（地图加载、协议处理、视口管理等）不做变动。

## 当前问题

1. **控件面板** — 无视觉层次，文件选择器是原生样式，checkbox 是原生方框
2. **调试面板** — 默认常驻右下角，遮挡地图且信息杂乱
3. **信息面板** — 纯等宽字体属性列表，无格式化
4. **进度条** — 3px 细线，几乎不可见
5. **配色** — 功能可用但缺乏精致感，玻璃拟态效果不明显
6. **无动画** — 面板显示/隐藏、悬停效果都是瞬时切换
7. **标题栏** — WebView 默认白色标题栏，与深色主题不协调
8. **底部状态栏** — 缺少坐标、缩放等实时信息的固定显示区

## 实施步骤

### 步骤 1：整体配色系统重构

重新定义 CSS 变量体系，建立统一的深色主题色板：

- `--bg-panel`: 面板背景 `rgba(15, 15, 35, 0.94)`
- `--border-glow`: 边框高光 `rgba(100, 140, 240, 0.25)`
- `--text-primary`: 主文字 `#e2e8f0`
- `--text-secondary`: 次要文字 `#8892a4`
- `--accent`: 强调色 `#60a5fa`
- `--accent-dim`: 弱化强调 `rgba(96, 165, 250, 0.15)`
- `--success` / `--error` / `--warn`: 状态色

所有面板统一 `backdrop-filter: blur(12px)` + 微妙的渐变边框。

### 步骤 2：控件面板重新设计

**文件选择区域**：用一个带图标的拖拽区域替代原生 `<input type="file">`，显示"点击选择文件夹或拖入"的提示。选中后显示文件夹名称和文件数量。

**VCM 开关**：将原生 checkbox 替换为 CSS 自定义滑动开关（toggle switch），带"等高线"文字标签。

**统计信息**：用图标+数值的卡片式布局替代纯文字，如"已索引 120 | 当前 24 | Z11.2"。

**面板标题**：左侧加一个小型地图图标（内联 SVG），标题字体改为半粗体 14px。

### 步骤 3：调试面板改进

- 默认隐藏，通过左下角的一个小圆形按钮（齿轮图标）切换显示
- 显示时半透明叠加在地图右下角，不遮挡控件面板
- 添加"清除日志"按钮
- 错误行用左侧红色竖线标记，而非仅靠文字颜色区分

### 步骤 4：信息面板（Feature Inspector）改进

- 添加标题栏：左侧"属性检查"标题，右侧关闭按钮（×）
- 属性用两列表格布局：左列属性名（紫色），右列值（绿色）
- 长文本值截断并可点击展开
- 添加一个"复制全部"按钮
- 面板出现/消失用 `opacity + transform` 过渡动画（200ms ease）

### 步骤 5：底部状态栏

在地图底部新增一条固定状态栏（高度 28px），实时显示：
- 左侧：鼠标经纬度坐标（`lat, lon`，保留 4 位小数）
- 中间：当前瓦片坐标（`z/x/y`）
- 右侧：数据加载状态（空闲时显示"Ready"，加载时显示进度）

状态栏样式：半透明深色背景，11px 等宽字体，不遮挡地图内容。

### 步骤 6：标题栏自定义

Tauri 窗口配置中设置 `decorations: false`（无边框窗口），在 HTML 中自绘标题栏：
- 高度 32px，可拖拽区域（`data-tauri-drag-region`）
- 左侧应用名 "MapViewer"
- 右侧最小化/最大化/关闭按钮

### 步骤 7：动画与过渡

- 面板显示/隐藏：`opacity 0→1` + `translateY(-8px→0)`，200ms
- 进度条：已有 `transition: width 0.15s`，增加颜色渐变效果
- 按钮/开关悬停：`background-color` 过渡 150ms
- 地图加载完成后面板淡入

### 步骤 8：进度指示器增强

- 进度条高度增加到 4px，加圆角和微弱发光效果
- 加载完成后进度条变为绿色并保持 1 秒后消失
- 扫描文件时在进度条右侧显示百分比数字

## 修改文件

只修改一个文件：`map-app/src/index.html`（CSS + HTML 结构）

## 验证方式

1. `cd map-app && npx tauri dev` 启动开发模式实时预览
2. 选择文件夹加载数据，确认所有功能正常
3. 检查各面板动画流畅度
4. 确认调试面板可切换隐藏/显示
5. 确认底部状态栏实时更新坐标
6. 确认 VCM 开关正常工作
7. 确认标题栏拖拽和窗口控制按钮正常
